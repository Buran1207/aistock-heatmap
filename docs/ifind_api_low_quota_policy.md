# iFind API 额度节约策略

## 总原则

系统默认“不为了刷新页面而浪费 API 额度”。所有能缓存的表都缓存，所有能增量更新的表都增量更新。

## 数据源分层

### 每天更新，但走低额度窗口

- 上市申请一览
- 首发信息一览
- 打新中签结果
- 孖展数据
- 基石投资者
- 限售股解禁明细
- 承销团 / 账簿管理人

### 每天增量更新

- 港股日行情

逻辑：本地已有行情到 T 日，则下一次默认补 T+1 到今天，同时保守补最近 5 天。

### 每天只拉当前池

- 收盘快照

逻辑：先由 IPO 主表生成 2024+ IPO 股票池，再只对这些股票拉快照。

## 不建议的做法

- 每天手工导出全港股主板日行情。
- 每天从 2024-01-01 全量重拉行情。
- 失败后无限重试。
- 把 iFind 账号密码写进代码或上传 GitHub。

## 失败处理

任何数据源失败时：

1. 保留旧缓存。
2. 记录到 `logs/update_YYYYMMDD.log`。
3. 写入 `deploy_data/data_update_status.csv`。
4. Streamlit 页面显示“失败，保留旧数据”。
